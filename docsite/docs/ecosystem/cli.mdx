# Komodo CLI

```mdx-code-block
import RemoteCodeFile from "@site/src/components/RemoteCodeFile";
```

The Komodo CLI, `km`, runs executions, updates resources and variables, opens server and container
terminals, and performs database backup or restore operations.

The Core container image also includes `km`. Running it inside the Core container lets the CLI use
the same database configuration as Core for database operations such as `km database backup -y`.

## Common Commands

```bash
km --help
km deploy stack my-stack
km run action my-action -y
km database backup
km db restore
km set var MY_VAR my_value -y
km update build my-build "version=1.19.0&branch=release"
km x commit my-sync
km set user mbecks super-admin true
km set user mbecks password "temp-password"
km ssh my-server
km connect my-server bash -n my-session
km exec my-container bash
km attach my-container -s my-server
```

## Terminals

All terminal commands share the same disconnect shortcut: press **Alt+Q** to end the session. The
session keeps running and preserves its history.

### SSH / Connect

`km ssh` is an alias for `km connect`. It opens an interactive shell on a server managed by
Komodo Periphery.

```text
km ssh <server> [command] [options]
km connect <server> [command] [options]
```

| Argument | Description |
| --- | --- |
| `server` | Server name. |
| `command` | Shell command to start, such as `bash`. Defaults to the Periphery default shell. |
| `-n`, `--name` | Terminal session name. Defaults to `ssh`. |
| `-r`, `--recreate` | Replace any existing session with the same name. |

### Exec

`km exec` opens an interactive shell inside a running container.

```text
km exec <container> <shell> [options]
```

| Argument | Description |
| --- | --- |
| `container` | Container name. |
| `shell` | Shell to use, such as `bash` or `sh`. |
| `-s`, `--server` | Server name when multiple servers have a container with the same name. |
| `-r`, `--recreate` | Replace any existing session with the same name. |

### Attach

`km attach` attaches to the stdio of a running container's main process.

```text
km attach <container> [options]
```

| Argument | Description |
| --- | --- |
| `container` | Container name. |
| `-s`, `--server` | Server name when multiple servers have a container with the same name. |
| `-r`, `--recreate` | Replace any existing session with the same name. |

## Install

CLI binaries are available for Linux (`x86_64`, `aarch64`) and macOS (`apple silicon`). A
distroless image is also published as `ghcr.io/moghtech/komodo-cli`.

### Linux

Install system-wide to `/usr/local/bin/km`:

```bash
curl -sSL https://raw.githubusercontent.com/moghtech/komodo/main/scripts/install-cli.py | python3
```

Install as the current user to `${HOME}/.local/bin/km`:

```bash
curl -sSL https://raw.githubusercontent.com/moghtech/komodo/main/scripts/install-cli.py \
  | python3 - --user
```

### macOS

Install through Homebrew:

```bash
brew tap moghtech/komodo
brew install km
```

### Container

Run the published CLI image:

```bash
alias km='docker run --rm -v $HOME/.config/komodo:/config ghcr.io/moghtech/komodo-cli:2 km'
km config
```

## Configuration

The CLI reads configuration for the Komodo host, API keys, database access, and profile selection.
CLI arguments override environment variables, and environment variables override file values.

To inspect the effective configuration, run:

```bash
km config
```

### File Detection

The CLI scans the current working directory and `${HOME}/.config/komodo` for files matching
`*komodo.cli*.*`. It parses them into one representation and merges them in discovery order.
Files found later override earlier values.

Use `-c` to pass one or more explicit config files:

```bash
km -c /path/to/base.config.yaml -c ./overrides.toml config
```

You can also add a `.kminclude` file inside one of the configured directories:

```text
# Supports comments

./.komodo
/etc/komodo/komodo.cli.toml
```

Wildcards are not supported in `.kminclude`.

### Profiles

Configuration files can define multiple profiles. Select one with `km -p <profile> ...`.
`default_profile` chooses the profile used when `-p` is omitted.

Top-level fields outside a profile act as shared defaults across profiles.

## Example Configuration File

The CLI also accepts YAML and JSON. These converters can help when you want another file format:

- [TOML to YAML](https://it-tools.tech/toml-to-yaml)
- [TOML to JSON](https://it-tools.tech/toml-to-json)

Download the sample file to `./komodo/komodo.cli.toml`:

```bash
wget -P komodo https://raw.githubusercontent.com/moghtech/komodo/main/config/komodo.cli.toml
```
